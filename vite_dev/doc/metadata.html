🏷️ Metadata Reference

Metadata in Mustela is defined within a meta do ... end block. These variables represent the state of your page and are accessible globally across your templates using the @ prefix.

1. Mutable Variables

These variables can be defined in your config/mustela.mu (globally) or overridden in specific .md / .mu files (locally).

Variable Description
title The title of the page/article.
description A short summary used for SEO and meta tags.
lang Defines the HTML language attribute (e.g., en, cs).
author The name of the content creator.
author_url A link to the author's profile or website.
base_url The root URL of the project, used for absolute path resolution.
index Router: Specifies which HTML block from your template to use as the entry point (defaults to main).
id_headers Toggle: If set to true, Mustela automatically generates id attributes for all Markdown headers.

2. Immutable Constants

These variables are automatically generated by the Mustela engine. You can read and display them, but you cannot overwrite them.

Constant System Description
view Core Placeholder: The most important constant. This is where the transformed Markdown content is injected into your template.
date_now Returns the current date and time formatted according to your global configuration.
filename Returns the name of the current file being processed (without the extension).

3. The Logic of index

The index variable acts as a template router. 

meta do
  index: article
end

In this example, Mustela will search your templates for a block named html as article do. If found, it uses that layout. If index is not defined, it defaults to main.

4. Header IDs (id_headers)

When building long-form documentation, you often need anchor links. By enabling id_headers, Mustela transforms your Markdown headers into linkable targets:

Markdown:

## 1. Installation

Output (if id_headers: true):

<h2 id="s1-installation">1. Installation</h2>

💡 Pro Tip

All metadata values are treated as Strings. Even boolean-like toggles like id_headers are parsed from their string representation.
